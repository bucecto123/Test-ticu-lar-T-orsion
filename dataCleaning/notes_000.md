To empirically validate the proposed neuro-symbolic architecture, the CIC-IDS2017 dataset was selected as the foundational evaluation environment. Specifically, the flow-aggregated MachineLearningCSV partitions were utilized rather than the raw .pcap packet captures.  This decision aligns with the architectural requirement to evaluate the "thermodynamic" and geometric properties of network behavior rather than inspecting deep packet payloads. By utilizing flow-level features engineered via CICFlowMeter, the network traffic is inherently compressed from highly unstructured byte streams into a structured 78-dimensional vector space, $\mathbb{R}^{78}$. This continuous/discrete hybrid space allows for the direct application of linear algebraic projections and abstract state machine validation without the prohibitive computational overhead of real-time payload parsing.

A core tenant of the proposed mathematical pipeline is the strict isolation of topological state transitions from continuous physical forces. To accommodate this, the 78-dimensional input space was bifurcated during the ingestion phase. Discrete TCP control flags (e.g., SYN, ACK, FIN, RST) were extracted to form an algebraic feature subset. This subset serves as the input for the deterministic Monoid state machine, which validates protocol compliance. The remaining continuous features—representing thermodynamic analogs such as Flow Bytes/s, Packet Length Variance, and Inter-Arrival Time (IAT)—were isolated to form the geometric subspace. This bifurcation ensures that categorical state markers do not mathematically distort the orthogonal projections calculated by the Geometric Engine.

The empirical dataset contains inherent computational artifacts that must be sanitized to ensure the stability of the downstream Singular Value Decomposition (SVD) algorithms. Notably, features derived from time-division calculations (such as Flow Packets/s) frequently manifest as Infinity or NaN values when a flow duration resolves to near-zero microseconds. Injecting mathematically undefined values into the Geometric Engine would instantly destabilize the calculation of the invariant subspace $P_U$. Consequently, a rigorous sanitization pipeline was implemented to actively identify and purge all non-finite numerical representations from the matrix prior to mathematical processing. Additionally, structural formatting errors, such as leading whitespaces in column headers, were programmatically stripped to ensure reliable tensor mapping.

To map the physical features into a proper Euclidean space suitable for residual magnitude calculations, two final preprocessing steps were applied. First, spatial overfitting was preemptively mitigated by the dataset's native omission of spatial identifiers (e.g., Source/Destination IP and MAC addresses). By removing network identities, the architecture is forced to evaluate anomalies based purely on behavioral physics rather than memorized topological locations. Second, to prevent high-magnitude features (e.g., Flow Duration measured in millions) from mathematically eclipsing low-magnitude features (e.g., Packet Length measured in hundreds) during orthogonal projection, the continuous geometric matrix was normalized. A Standard Scaler was applied to ensure all continuous dimensions achieved a mean of zero and unit variance, establishing a balanced vector space for accurate anomaly detection.
